What it is

A map for agents working across languages and tools.

Thule gives AI coding agents a compact, tool-native set of instructions for real engineering work. Each skill tells the agent how to detect project conventions, prefer existing scripts, run appropriate checks, and report risks clearly.

Why the name

Thule points to the edge of the known map.

In classical geography, Thule evoked the far northern boundary of the known world. For this project, the name is used in that ancient-map sense: a guidepost for agents navigating the edge between human engineering judgment and automated coding assistance.

The image comes from Qaanaaq, Greenland, historically called New Thule. It anchors the metaphor in a real place while keeping the project focused on practical developer guidance.

Install

Choose a tool, choose a language, install only what you need.

./install.sh ./install.sh --tool codex --lang go --scope personal ./install.sh --tool cursor --lang typescript --scope project --target /path/to/repo

Future roadmap

Next, make the map richer without making it heavier.

  1. Expand language coverage

    Add Kotlin, C#, C/C++, Ruby, PHP, and Swift with the same official-docs-first standard.

  2. Add framework-specific packs

    Create optional skills for Spring, Akka, React, Next.js, FastAPI, Django, Axum, and Tokio.

  3. Strengthen scanner workflows

    Add references for Semgrep, CodeQL, Trivy, osv-scanner, cargo-deny, and ecosystem-native audit tools.

  4. Improve installer ergonomics

    Add update, uninstall, diff, and version pinning commands for repeatable team installs.

  5. Publish examples

    Show before-and-after agent runs so teams can see how each skill changes behavior in real repositories.

Thule is built for programmers using AI agents to work carefully across many languages.
